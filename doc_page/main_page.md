Practical Work 1 - RailWay Managment

‎ ‎ ‎ ‎

The "Railway Management" work is an implementation of a Graph class that
represents a railway network management system. The Graph class has
methods to add stations to the network, add connections between
stations with service capacities, remove connections between stations,
add stations to specific lines, perform shortest path algorithms
(Dijkstra), and calculate maximum flow (Edmonds-Karp algorithm) 
between stations. Additionally, the class has methods to retrieve 
information about stations, lines, maximum flow in districts and 
municipalities, and calculate the maximum capacity of trains that 
can be scheduled at a station.

‎ ‎ ‎ ‎

When you choose one of the options from the menu displayed in the code snippet you provided, the program will execute the corresponding functionality associated with that option. Here is a brief overview of each option:

1. Specific Station Information: This option will display information about a specific station in the railway network, such as its name, location, and other relevant details.

2. Maximum number of trains that can simultaneously travel between two stations: This option will calculate and display the maximum number of trains that can travel simultaneously between two specified stations in the railway network, taking into account their service capacities.

3. Shortest path between two stations: This option will calculate and display the shortest path between two specified stations in the railway network, using the Dijkstra algorithm or another suitable algorithm for finding the shortest path in a graph.

4. Display the pair of stations that needs trains: This option will display the pair of stations in the railway network that require trains based on certain criteria, such as high demand or other predefined conditions.

5. Top-k municipalities/districts with highest budget needs: This option will display the top-k municipalities or districts in the railway network that have the highest budget needs, based on certain criteria such as budget allocation or expenditure.

6. Max number of trains that can arrive at a train simultaneously: This option will calculate and display the maximum number of trains that can arrive simultaneously at a specific station in the railway network, taking into account the station's capacity and other relevant factors.

7. Maximum concurrent trains with minimal cost for the company between two stations: This option will calculate and display the maximum number of concurrent trains that can travel between two specified stations in the railway network, while minimizing the cost for the company based on predefined cost factors.

8. Quit: This option will exit the menu loop and terminate the program.
